# 已知图库Schema

## 节点类型

{node_schema}

## 关系类型

{relation_schema}

# 任务说明

根据上述图库结构分析用户的查询信息

# 请识别以下内容

1. 查询类型（query_type）
   - entity_relation：查询两个实体之间的直接关系，如：某公告的发布单位是谁？
   - multi_hop：需要多跳推理，如：某标准规范涉及哪些术语定义？
   - subgraph：需要获取某一实体相关的完整子图，如：某标准规范的全部相关信息
   - path_finding：路径查找，如：标准规范 → 文档 → 公告 → 发布机构
   - clustering：相似内容/结构查询，如：类似某标准的规范有哪些？

2. source_entities（起点实体）
   - 只包含图中可能存在的具体节点名称
   - 优先包括：
     - 标准名称（如："公路工程技术标准"）
     - 文档名称
     - 公告编号/标题
     - 术语名称
     - 符号（如：E₀）
     - 组织机构名称
     - 人员姓名

3. target_entities（目标实体）
   - 仅在明确需要"终点约束"时填写
   - 例如：
     - "术语"
     - "公式"
     - "图片"
     - "组织机构"
     - "人员"
   - 不确定时返回空数组 []

4. relation_types（关系类型）
   从 Schema 中选择最相关的关系，例如：
   - 文档内容类："关联公式"、"关联图片"、"关联表格"、"关联代码"
   - 知识类："术语"、"符号"
   - 标准体系："关联文档"
   - 公告类："关联公告"、"发文机关"、"印发机关"
   - 人员/组织："主编"、"主审"、"参编单位"、"主编单位"、"联系人"
   - 结构类："关联目录"、"子目录项"

5. max_depth（最大跳数）
   - 1：单跳查询（如 文档 → 术语）
   - 2：常规关联（如 标准 → 文档 → 术语）
   - 3：复杂路径（如 标准 → 文档 → 公告 → 机构）

6. constraints（属性约束）
   用于表达非结构化过滤条件：

   ```json
   {{
     "时间": {{
       "发布日期": "2020-01-01之后"
     }},
     "类型": ["强制性标准"],
     "领域": ["公路工程"],
     "关键词": ["承载力", "变形模量"]
   }}
   ```

   注意：不要包含抽象条件（如："最新的"、"重要的"、"详细说明"）

# 输出要求

请严格返回一个合法 JSON：

- 不要添加解释
- 不要输出多余文本
- 所有字段必须存在（没有则用空值）

```json
{{
  "query_type": "",
  "source_entities": [],
  "target_entities": [],
  "relation_types": [],
  "max_depth": 1,
  "constraints": {{}}
}}
```

# 示例

## 示例1

- 查询：
  某标准规范中有哪些术语定义？
- 分析：
  从“标准规范”出发
  通过“文档”再到“术语”
  属于 multi_hop
- 返回：
  ```json
  {{
    "query_type": "multi_hop",
    "source_entities": ["某标准规范"],
    "target_entities": ["术语"],
    "relation_types": ["关联文档", "术语"],
    "max_depth": 2,
    "constraints": {{}}
  }}
  ```

## 示例2

- 查询：
  某公告的发布机构是谁？
- 分析：
  公告 → 发文机关
  单跳 entity_relation
- 返回：
  ```json
  {{
    "query_type": "entity_relation",
    "source_entities": ["某公告"],
    "target_entities": ["组织机构"],
    "relation_types": ["发文机关"],
    "max_depth": 1,
    "constraints": {{}}
  }}
  ```

## 示例3

- 查询：
  “公路工程标准中2020年后发布的有哪些？”
- 分析：
  标准规范节点
  时间过滤属于 constraints
- 返回：
  ```json
  {{
    "query_type": "subgraph",
    "source_entities": ["公路工程标准"],
    "target_entities": [],
    "relation_types": ["关联文档"],
    "max_depth": 1,
    "constraints": {{
      "时间": {{
        "发布日期": "2020-01-01之后"
      }}
    }}
  }}
  ```
