# 任务说明

根据上述图库结构分析用户的查询信息

# 请识别以下内容

1. 查询类型（query_type）
   - entity_relation：查询两个实体之间的直接关系，如：某公告的发布单位是谁？
   - multi_hop：需要多跳推理，如：某标准规范涉及哪些术语定义？
   - subgraph：需要获取某一实体相关的完整子图，如：某标准规范的全部相关信息
   - path_finding：路径查找，如：标准规范 → 文档 → 公告 → 发布机构
   - clustering：相似内容/结构查询，如：类似某标准的规范有哪些？
   - entity_query: 查询实体信息，如：某个术语定义信息

2. source_entities（起点实体）
   - 格式：
    ```json
    [{{
      "label": "<节点标签>",
      "constraints": {{
        "<节点属性>": "<节点属性值>"
      }}
    }}]
    ```
   - 说明：
     - label: 必填，必须是Schema中定义的节点类型，例如：
       - 标准规范
       - 文档
       - 公告
       - 术语
       - 符号
       - 人员
       - 组织机构
     - constraints: 可选，用于限定具体实体（推荐使用），例如：
       ```json
       {{
         "姓名": "张志新"
       }}
       ```
     - 若用户问题中包含明确实体名称，必须写入constraints中
   - 示例：
     - 查询姓名为“张志新”的人员： 
     ```json
     [{{"label": "人员", "constraints": {{"姓名": "张志新"}}}}]
     ```

3. target_entities（目标实体）
   - 格式：
    ```json
    [{{
      "label": "<节点标签>",
      "constraints": {{
        "<节点属性>": "<节点属性值>"
      }}
    }}]
    ```
   - 说明：
       - label：必填
       - constraints：可选
       - 仅在存在明确“查询目标类型”时填写
   - 示例：
       - 查询术语：
       ```json
       [{{"label": "术语"}}]
       ```
       - 查询机构：
       ```json
       [{{"label": "组织机构"}}]
       ```
   - 不确定时返回空数组 []

4. relation_types（关系类型）
   从 Schema 中选择最相关的关系，例如：
   - 文档内容类："关联公式"、"关联图片"、"关联表格"、"关联代码"
   - 知识类："术语"、"符号"
   - 标准体系："关联文档"
   - 公告类："关联公告"、"发文机关"、"印发机关"
   - 人员/组织："主编"、"主审"、"参编单位"、"主编单位"、"联系人"
   - 结构类："关联目录"、"子目录项"

5. max_depth（最大跳数）
   - 1：单跳查询（如 文档 → 术语）
   - 2：常规关联（如 标准 → 文档 → 术语）
   - 3：复杂路径（如 标准 → 文档 → 公告 → 机构）

6. constraints（属性约束）
   用于表达非结构化过滤条件：

   ```json
   {{
     "时间": {{
       "发布日期": "2020-01-01之后"
     }},
     "类型": ["强制性标准"],
     "领域": ["公路工程"],
     "关键词": ["承载力", "变形模量"]
   }}
   ```

   注意：不要包含抽象条件（如："最新的"、"重要的"、"详细说明"）

# 输出要求

请严格返回一个合法 JSON：

- 不要添加解释
- 不要输出多余文本
- 所有字段必须存在（没有则用空值）

```json
{{
  "query_type": "",
  "source_entities": [],
  "target_entities": [],
  "relation_types": [],
  "max_depth": 1,
  "constraints": {{}}
}}
```

# 示例

## 示例1

- 查询：
  张志新参与了哪些规范制定?
- 分析：
  从“人员”出发
  通过“文档”再到“标准规范”
  属于 multi_hop
- 返回：
  ```json
  {{
    "query_type": "multi_hop",
    "source_entities": [{{"label": "人员", "constraints": {{"姓名": "张志新"}}}}],
    "target_entities": [{{"label": "标准规范"}}],
    "relation_types": ["主编", "主审", "主要参编人员", "参加人员", "参审人员", "关联文档"],
    "max_depth": 2,
    "constraints": {{}}
  }}
  ```

  文档 → 组织机构

  ```json
    "query_type": "entity_relation",

    "target_entities": ["组织机构"],
    "relation_types": ["参编单位"],
    "max_depth": 1,
    "constraints": {{}}
  }}
  ```
